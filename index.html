🔢 Step 2 — Min-Max Normalization▼

Because TPH, CSAT, and IQS use different scales, each KPI is normalized to a 0–1 range before combining:

Normalized = (Agent Value − Team Min) / (Team Max − Team Min)

An agent at the team minimum gets 0.0; the top agent gets 1.0. All others are proportional in between. This prevents a KPI with larger absolute numbers (e.g., CSAT%) from dominating a KPI with smaller numbers (e.g., TPH).

🏆 Step 3 — Weighted Composite Score▼
Composite = (TPH_norm × 0.40) + (CSAT_norm × 0.40) + (IQS_norm × 0.20)

Weights reflect business priorities: CSAT and TPH are equally important (40% each) as they directly measure customer impact and output. IQS is secondary (20%) — process compliance matters, but shouldn't penalize an agent who is highly productive and customer-positive.

📐 Step 4 — Two-Stage Quartile Assignment▼

Stage A — Initial placement: Rank agents by Composite Score (descending). Assign by relative rank: top 25% → Q1, 26–50% → Q2, 51–75% → Q3, bottom 25% → Q4.

Stage B — Worst KPI Override: Check each agent's worst normalized KPI score:

  • < 0.25 (bottom quartile range): Agent cannot be placed higher than Q3, regardless of composite. Flag: "OVERRIDE — Worst KPI drag"
  • < 0.10 (critical underperformance): Agent placed directly in Q4. Flag: "OVERRIDE — Critical KPI gap"

Why overrides? Without this, an agent could score well in two KPIs and mask a critical failure in the third. The override ensures no coaching gap is hidden.

Tie-handling: Identical composite scores at a boundary → both assigned to the lower quartile (conservative coaching bias).

Non-divisible teams: Extra agents go to Q2 (middle-up bias — under-coaching a borderline agent is preferable to overloading supervisors).

🚀 Share with Your Team — GitHub Pages Deployment

Host this page for free on GitHub Pages so anyone on your team can access it via a shared link — no installation needed.

1
Create a GitHub account

Go to github.com and sign up for a free account if you don't have one.

2
Create a new repository

Click + → New repository. Name it e.g. cx-quartilization. Set it to Public. Click Create repository.

3
Upload index.html

In your new repo, click Add file → Upload files. Drag and drop index.html. Commit the file.

4
Enable GitHub Pages

Go to Settings → Pages. Under Source, select Deploy from branch → main → / (root). Click Save.

5
Share the link

After ~1 minute your page will be live at: https://YOUR-USERNAME.github.io/cx-quartilization/. Share this URL with your team. Each person logs in with their own name/email.
